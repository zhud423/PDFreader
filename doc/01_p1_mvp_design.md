# P1 自用 MVP 详细方案

## 1. 目标

P1 只解决一件事：让用户可以在 `iPhone / iPad` 上，把 PDFreader 当成一个可自用的长竖页 PDF 漫画阅读器开始使用。

核心闭环：

`手动选择 PDF -> 打开阅读 -> 顺滑上下滑动 -> 退出 -> 下次看到继续阅读`

## 2. 成功标准

### 2.1 功能成功标准

- 可以导入单个或多个 PDF
- 可以自动生成封面
- 可以进入阅读器并顺滑阅读长竖页 PDF
- 可以记录每本书的精确阅读位置
- 首页可以显示上次阅读的书
- 当源文件不可再次访问时，条目仍保留，但显示不可用

### 2.2 体验成功标准

- 首次打开大文件允许稍慢，但不应阻塞到无法使用
- 阅读阶段滚动和缩放优先保证流畅
- 沉浸阅读干扰最小
- 用户不需要学习复杂操作

## 3. 范围定义

## 3.1 P1 要做

- PWA 安装基础能力
- 本地手动上传 PDF
- 首页继续阅读卡片
- 简化书籍列表
- PDF 首页自动截封面
- 长竖页手动上下滑动阅读
- 基础双指缩放
- 横竖屏适配
- 单击显示 / 隐藏工具栏
- 自动隐藏操作区
- 记录上次阅读书籍
- 记录每本书的精确滚动位置
- 缺失文件显示不可用并支持重新选择

## 3.2 P1 不做

- 完整书库管理
- 搜索
- 排序
- 收藏
- 在读 / 已读状态
- 目录 / 缩略图导航
- 快速跳页
- 自动滚动
- 图像增强
- 多设备同步

## 4. 关键产品约束

### 4.1 不存 PDF 副本

P1 不会把原始 PDF 存进应用本地空间，只保存以下内容：

- 书籍元数据
- 封面缩略图
- 阅读进度
- 来源信息

这符合“文件很大，不占设备空间”的目标，但也意味着本地文件的可再次访问能力受浏览器限制。

### 4.2 本地文件的继续阅读存在平台限制

对 `iPhone / iPad Safari` 而言，PWA 不能把本地上传文件当作一个长期稳定可重开的数据源来依赖。

P1 的设计结论：

- 进度和条目会永久保留
- 如果浏览器进程丢失了对源文件的访问，书籍会显示为“不可用”
- 用户可以重新选择原文件后恢复阅读
- 真正稳定的一键继续阅读，会在 `P2` 的局域网 URL 书源引入后显著改善

### 4.3 长竖页是主场景

样例漫画是典型的长竖页 PDF，因此阅读器默认按以下方式优化：

- 单页长图顺序阅读
- 纵向滚动优先
- 不做目录化分页体验
- 缩放存在，但不打断纵向阅读为第一目标

## 5. 用户故事

### 5.1 首次使用

用户第一次打开应用后，可以看到产品说明和导入入口。选择一个 PDF 后，应用解析封面和基础元数据，并进入阅读器。

### 5.2 继续阅读

用户再次打开应用后，首页展示“继续阅读”卡片。如果来源仍可访问，则一键回到上次阅读位置；如果来源不可访问，则展示不可用状态与“重新选择文件”入口。

### 5.3 查看已导入书籍

用户可以在首页下方看到已导入书籍列表，包含封面、标题、上次阅读时间、可用状态。

### 5.4 阅读与退出

用户进入阅读页后默认沉浸式阅读，单击切换工具栏显隐，退出后自动保存当前位置。

## 6. 信息架构

P1 只保留两个主页面和一个导入流程：

- 首页
- 阅读页
- 导入动作层

### 6.1 首页

模块建议：

- 顶部品牌区
- 继续阅读卡片
- 导入按钮
- 最近导入 / 最近阅读列表
- 缺失文件条目提示

### 6.2 阅读页

模块建议：

- 顶部浮层工具栏
- 阅读画布区
- 底部极简状态栏
- 错误与加载提示层

## 7. 页面与交互设计

## 7.1 首页

### 布局

- 首屏优先展示“继续阅读”
- 导入入口固定在首屏可见区域
- 已导入书籍列表可纵向滚动

### 关键元素

- `继续阅读卡片`
  - 封面
  - 书名
  - 上次阅读时间
  - 进度摘要
  - 可用 / 不可用状态
- `导入按钮`
  - 支持单个与多个 PDF
- `最近书籍列表`
  - 书名
  - 封面
  - 状态标签

### 状态

- 空状态：引导导入第一本 PDF
- 有书但无可继续阅读：显示最近书籍
- 继续阅读可用：直接可点开
- 条目不可用：显示“源文件不可访问”

## 7.2 导入流程

### 交互步骤

1. 用户点击导入
2. 系统打开文件选择器
3. 用户选择一个或多个 PDF
4. 应用读取基础元数据
5. 截取第一页低分辨率封面
6. 写入书库记录
7. 进入所选书籍阅读页

### 导入时要做的事

- 校验文件类型
- 计算书籍唯一标识
- 获取页数和第一页尺寸
- 生成封面缩略图
- 建立初始进度记录

## 7.3 阅读页

### 默认状态

- 进入即沉浸式
- 顶部和底部操作区短时间后自动隐藏
- 内容区占满可用空间

### 交互规则

- 单击内容区：切换工具栏显隐
- 双指操作：基础缩放
- 纵向拖动：手动上下滑动阅读
- 返回：首页并自动保存当前位置

### 工具栏建议

顶部工具栏：

- 返回
- 标题
- 当前状态提示

底部状态栏：

- 当前阅读进度摘要
- 加载状态

P1 不做复杂菜单，避免破坏沉浸感。

## 7.4 不可用条目恢复

当条目来源不可访问时：

- 首页继续阅读卡片显示不可用
- 点击后优先弹出“重新选择文件”
- 重新选择成功后，保留原进度并继续阅读

## 8. 数据模型

P1 建议使用 `IndexedDB + Dexie`。

## 8.1 books

- `id`
- `title`
- `displayTitle`
- `sourceType`：`local_upload`
- `sourceKey`
- `fileName`
- `fileSize`
- `pageCount`
- `firstPageWidth`
- `firstPageHeight`
- `coverBlobKey`
- `availabilityStatus`
- `createdAt`
- `updatedAt`
- `lastOpenedAt`

## 8.2 progress

- `bookId`
- `lastPageIndex`
- `scrollOffset`
- `zoomScale`
- `viewportWidth`
- `viewportHeight`
- `updatedAt`

P1 必须至少稳定保存 `scrollOffset`，`zoomScale` 可以先存储但不承诺完整恢复手感。

## 8.3 app_state

- `lastOpenSourceContext`

## 8.4 cover_cache

- `bookId`
- `blob`
- `width`
- `height`
- `createdAt`

## 9. 技术设计

## 9.1 前端栈

- `React`
- `TypeScript`
- `Vite`
- `pdf.js`
- `Dexie`
- `vite-plugin-pwa`

## 9.2 阅读器核心设计

### 文档加载

- 使用 `pdf.js` worker 解析 PDF
- 先读取元数据与第一页信息
- 封面生成使用低分辨率渲染，避免导入阶段过慢

### 渲染策略

- 不一次性把整页渲染到单个超大 canvas
- 按纵向分段渲染长竖页
- 仅保留可视区域附近的段
- 离开视口较远的段及时释放位图 / canvas 资源

### 进度恢复

- 阅读过程中节流保存位置
- 页面隐藏、切后台、返回首页时强制保存
- 恢复时优先回到上次段落与滚动位置

## 9.3 性能策略

- 首屏只解析必要信息
- 封面使用低清图
- 阅读时启用懒加载
- 相邻分段预加载
- 缩放时降低重渲染频率，优先保持交互平滑

## 10. 非功能要求

### 10.1 性能

- 大文件首开允许较慢，但应可见加载反馈
- 阅读过程应以滚动稳定性为最高优先级
- 内存压力增大时，优先丢弃远离视口的渲染缓存

### 10.2 容错

- 导入失败时给出明确错误提示
- 条目失效时不丢进度
- PDF 解析异常时允许重新导入

### 10.3 离线

- 应用壳离线可打开
- 书籍元数据与封面离线可见
- 已失去文件访问能力的本地书籍，显示为不可用

## 11. 风险与对应策略

### 风险 1：iOS 本地文件句柄不稳定

策略：

- P1 不承诺“本地上传后永久一键直开”
- 用不可用状态与重新关联承接
- P2 引入局域网 URL 书源改善体验

### 风险 2：超长页渲染占用过大

策略：

- 分段渲染
- 低分辨率封面
- 可视区附近缓存
- 远离视区资源及时释放

### 风险 3：缩放导致卡顿

策略：

- P1 只做基础缩放
- 高级缩放体验放到 P2 优化

## 12. 验收标准

- 用户能在 iPhone / iPad 上安装 PWA
- 用户能导入至少一个大体积 PDF 并进入阅读
- 阅读页能完成顺滑纵向阅读
- 首页能展示继续阅读信息
- 关闭应用后，进度和书籍条目不丢失
- 当源文件无法继续访问时，条目保留且状态正确

## 13. 开发顺序建议

1. 项目脚手架与 PWA 基础壳
2. IndexedDB 数据层
3. PDF 导入与封面生成
4. 首页继续阅读与列表
5. 阅读器基础版
6. 进度保存与恢复
7. 不可用状态与重新关联
8. iPhone / iPad 适配与性能收尾
